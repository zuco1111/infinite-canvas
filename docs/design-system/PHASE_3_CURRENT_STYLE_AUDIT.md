# 设计系统阶段 3：现有样式对比审计

状态：进行中；用户已于 2026-07-12 批准开始阶段 3，当前只执行只读现有样式审计。

启动日期：2026-07-12

## 1. 用户目标

将相似的现有样式放在一起，使用户可以在同一视野中直接比较差异并做后续判断。

本切片不创建目标 Token、目标组件或候选视觉，不修改生产组件、Token、页面视觉或冻结基线，也不因为外观相似而推断合并关系。

## 2. 审计工具

Design Lab 新增两个查看方式：

- `相似项对比`：默认视图，按审计用途把相似项双列并置。
- `库存顺序`：保留阶段 2 的 Foundations → Primitives 稳定顺序，继续用于逐项追踪。

两个视图共享原 40 个库存 ID、稳定锚点、真实 App Provider、Light/Dark、桌面/移动视口、搜索和来源证据。新增“对比组”筛选；每个对比卡片默认突出当前实现，来源、使用范围、解析值和决策状态收纳在可展开证据区。

## 4. 边界与风险

- “相似”只表示适合并置评审，不表示语义、行为、键盘契约或可访问性相同。
- 完整 Patterns 和 Feature Components 仍按长期计划留到阶段 5，不在当前 Primitive 审计中复制。
- 首页非导航视觉冻结规则继续生效；当前切片不触碰生产视觉。
- DS-O1、DS-O6～DS-O10、DS-O13、DS-O15 仍开放。它们不阻塞本次只读审计，但继续阻塞候选 Token 事实源、目标值、正式可访问性目标和迁移期限。
- 当前截图只能支持视觉与可见结构判断，不能单独证明键盘、读屏、200% 缩放或 WCAG 合规。

## 6. 本次审计结论

### 6.1 整体判断

当前 10 个对比组足以把主要相似项放进同一视野；桌面双列和移动单列均没有内容横向溢出。阶段 2 稳定库存顺序仍可恢复，因此新编排没有破坏旧证据定位。

### 6.2 可见优势

- 主题/颜色的多事实源首次在一个连续组中出现，App、CSS、Canvas、Ant、裸值和遗留项可直接横向比较。
- 输入族与选择族不再被 Primitive 编号打散，真实 Ant、Radix、Native 和 Shared UI 差异更容易判断。
- 每个组都有明确审计重点，并显式说明“并置不等于合并”；来源证据可按需展开，默认视图更聚焦当前外观。

### 6.3 风险与证据限制

- 当前 `PRM-C10` 可见两个无可访问名称的 Ant Slider，`PRM-C14` 可见三个无可访问名称的 Ant Progress；这是现有生产实现风险，本切片不修复。
- 相似性分组不能证明语义、键盘契约或读屏输出相同，不能直接转成 merge/variant 关系。
- 截图不能证明 200% 缩放、完整键盘路径、读屏或 WCAG 合规；DS-O13 继续开放。
