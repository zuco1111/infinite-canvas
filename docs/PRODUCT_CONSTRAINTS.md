# 产品约束

## 视觉与行为

- 未经用户批准，不进行视觉 redesign。
- 等值重构必须保持既有功能行为和视觉输出一致。
- 设计 Token 和可复用组件抽取只能做等值抽取，不得借机调整视觉。
- 旧项目如存在已知或审计中发现的问题，先记录为后续修复项；除非用户明确批准，不在行为等效重构阶段修复。

## 提示词数据源

- 提示词库运行时数据库使用本地静态数据 `public/data/local-prompts.json`。
- 当前随包内置静态提示词数据已一次性清理无有效封面图的条目，包括空 `coverUrl` 和只指向语言徽章的 shields.io URL；首页、提示词库页面和提示词 API 不得在运行时按封面图过滤提示词。
- 用户自建或本地保存的提示词可以没有封面图，不得因为缺少 `coverUrl` 被过滤、拒绝保存或删除。
- 不得在未确认的情况下重新引入运行时外部提示词数据库抓取。
- 外部来源数据如需更新，应先离线验证可访问性，移除 404 来源后再写入本地提示词数据文件。

## 生成用量展示

- 当前业务不包含生成用量计量体系。
- 生成入口、模型配置和错误文案不得展示用量扣减、余额、购买套餐或闪电用量标识，除非用户后续明确批准。
- 已确认例外：设置弹窗渠道提示可展示“使用前请先配置可用 API 渠道。没有API？”和“立即购买”入口，链接指向 `https://api.zuco.ai/`。

## WebDAV 同步

- 当前目标项目使用直接 WebDAV 请求路径。
- Vite 目标项目未保留旧 Next.js route handler 形式的 WebDAV proxy；不得重新引入 `/webdav-proxy` 或 `nextjs` proxy 模式，除非用户后续明确批准新的平台 adapter 或独立服务方案。

## 数据兼容

- 本仓库不需要兼容旧项目已有数据、文件格式、本地缓存或历史项目数据。
- Phase 8 已确认不需要兼容本仓库当前本地持久化数据。
- 若后续需要重新引入历史兼容、旧字段 alias、旧 API fallback、旧 storage key 或旧样式别名，必须先记录原因并获得用户确认。
